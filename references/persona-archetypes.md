# 페르소나 아키타입 가이드

이 문서는 검토 페르소나 4~5명을 만들 때 참고하는 **메뉴**다. 그대로 복사하지
말고, 실제 문서의 주제·도메인·목적에 맞게 구체화한다.

## 핵심 원칙

1. **문서에서 도출한다.** 같은 "기획서"라도 B2B SaaS와 학원 운영은 다른
   페르소나가 필요하다. 아래 목록은 출발점일 뿐이다.
2. **상호 보완 + 충돌.** 4~5명의 관심사가 서로 다르고, 일부는 서로 부딪혀야
   한다 (예: "비용 절감" vs "사용자 경험", "출시 속도" vs "품질"). 충돌이
   없으면 Round 2 반박이 공허해진다.
3. **성향을 분산한다.** 전원 비판적이면 단조롭다. 권장 배합:
   - 1명 이상 **비판적** (날카롭게 문제를 판다)
   - 1명 이상 **지지적** (강점을 옹호하고 과한 비판을 견제)
   - 나머지는 **실용적 / 비전형 / 디테일형**으로 섞는다
4. **이름을 준다.** 회의 발언자 식별을 위해 실제 사람 이름을 붙인다.
5. **결정권자를 1명 포함**한다. 문서의 운명을 좌우할 위치의 사람(경영진/
   투자자/고객 등)이 있어야 회의가 현실적이다.

## 페르소나 필드 (Step 1 참조)

이름 · 역할/직함 · 전문 분야 · 검토 관점 · 핵심 관심사(2~3) · 성향

## 문서 유형별 추천 검토 관점

### 기획서 / PRD / 제품 요구사항
- **최종 사용자 대변자** — 실제 쓸 사람 입장. "이게 내 문제를 푸는가?"
- **엔지니어 (실현 가능성)** — 기술적으로 만들 수 있는가, 숨은 복잡도
- **비즈니스 / ROI** — 돈이 되는가, 우선순위가 맞는가, 기회비용
- **디자이너 / UX** — 사용 흐름, 직관성, 접근성
- **리스크 / 법무 / 보안** — 규제·개인정보·실패 시나리오

### 제안서 / 사업계획서
- **투자자 / 의사결정자** — 설득력 있는가, 숫자가 믿기는가
- **잠재 고객** — 내가 이걸 살 이유가 있는가
- **재무** — 추정의 근거, 현금흐름, 손익 구조
- **경쟁 분석가** — 차별점, 경쟁사 대비 우위가 진짜인가
- **실행 책임자** — 이 계획을 실제로 굴릴 수 있는가

### 보고서 / 분석 자료
- **경영진 (의사결정자)** — 결론이 명확하고 행동 가능한가
- **데이터 검증자** — 수치·출처·논리의 정확성
- **현장 실무자** — 현실과 맞는가, 누락된 맥락
- **외부 이해관계자** — 제3자가 봐도 공정·투명한가

### 계약서 / 법적 문서
- **법무** — 조항의 위험, 모호한 표현, 누락된 보호장치
- **재무** — 지급 조건, 비용, 페널티
- **사업 담당자** — 실무상 이행 가능한 조건인가
- **리스크 관리자** — 최악의 시나리오, 분쟁 가능성

### 마케팅 카피 / 홍보 문안
- **타겟 고객** — 마음이 움직이는가, 와닿는가
- **브랜드 매니저** — 브랜드 톤·일관성에 맞는가
- **카피라이터** — 문장력, 후킹, 군더더기
- **법무 (과장광고)** — 표현이 사실이고 법적으로 안전한가
- **퍼포먼스 마케터** — 전환·CTA·측정 가능성

### 기술 스펙 / 설계 문서 / RFC
- **시니어 엔지니어 / 아키텍트** — 설계의 견고함, 확장성, 트레이드오프
- **보안** — 공격 표면, 데이터 보호
- **운영 / SRE** — 배포·모니터링·장애 대응
- **주니어 (가독성)** — 처음 보는 사람이 이해할 수 있는가
- **인접 팀** — 의존성·인터페이스 영향

### 정책 / 프로세스 / 가이드 문서
- **적용 대상 (현장 직원)** — 실제로 지킬 수 있는가, 부담은
- **관리자** — 운영·집행이 가능한가
- **컴플라이언스 / 법무** — 규정 정합성
- **HR / 조직 문화** — 형평성, 사기 영향

### 학술 / 연구 / 교육 자료
- **해당 분야 전문가** — 내용의 정확성·깊이
- **대상 학습자 / 독자** — 난이도, 이해 가능성
- **편집자** — 구조, 흐름, 명료성
- **회의론자** — 주장의 빈틈, 반례

## 문서 유형이 애매할 때

위 분류에 딱 안 맞으면, 다음 4개 보편 관점으로 시작해 도메인에 맞게 변형한다:
- **대상 독자 대변자** — 이 문서를 받을 사람
- **비판적 검증자** — 약점·빈틈을 찾는 사람
- **실행/실무 관점** — 이대로 실행 가능한지 보는 사람
- **결정권자** — 이 문서로 무언가를 결정할 사람

5번째는 문서의 가장 큰 리스크 영역(법무/기술/재무/윤리 등)에서 고른다.
